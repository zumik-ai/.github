## Why Zumik?

AI agents repeatedly send the same context:

- System prompts
- Tool definitions
- Schemas
- Repository instructions
- Long conversation history
- Retrieved knowledge

Even with default caching with providers, over time, those costs add up.

Zumik turns that into reusable state, reducing costs, latency, and complexity

## Features

- **OpenAI-compatible API** (`/v1`)

- **Stateful agent sessions** (`/v2`)

- **Workload reuse diagnostics**

- **Multi-provider routing**

- **Latency & cost optimization**
